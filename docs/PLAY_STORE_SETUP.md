## 2. Política de privacidad

### URL de la política de privacidad
**IMPORTANTE:** Debes crear y publicar una política de privacidad en línea.

**Opciones:**
1. Crear una página en tu sitio web
2. Usar GitHub Pages (gratuito)
3. Usar servicios como TermsFeed o FreePrivacyPolicy

**Contenido mínimo requerido:**
- Qué datos recopilas (email, fotos, mensajes, ubicación de red)
- Cómo usas los datos
- Con quién compartes los datos (Supabase, Google AdMob)
- Cómo proteges los datos
- Derechos del usuario (acceso, eliminación)
- Contacto para consultas

**Ejemplo de URL:**
```
https://tupagina.com/moodpareja/privacy-policy
```

**Categoría:** Redes sociales / Comunicación

**Clasificación esperada:**

## 7. Seguridad de los datos

### Datos recopilados y compartidos

#### **Información personal**
```
☑️ Nombre
☑️ Dirección de correo electrónico
☑️ Fotos y videos
☑️ Mensajes (chat y voz)
```

#### **Información de la app**
```
☑️ Registros de fallos
☑️ Diagnósticos
☑️ Datos de rendimiento
```

#### **Información del dispositivo**
```
☑️ ID del dispositivo
☑️ Estado de la red
```

### ¿Cómo se usan los datos?

**Funcionalidad de la app:**
- Email: Autenticación y recuperación de cuenta
- Fotos: Compartir imágenes privadas con la pareja
- Mensajes: Comunicación entre parejas
- Datos del dispositivo: Notificaciones push

**Análisis:**
- Diagnósticos y registros de fallos para mejorar la app

**Publicidad:**
- ID del dispositivo para personalización de anuncios (AdMob)

### ¿Los datos se comparten con terceros?

**SÍ** - Con los siguientes servicios:

1. **Supabase** (Backend y almacenamiento)
   - Datos compartidos: Email, fotos, mensajes, perfil
   - Propósito: Funcionalidad de la app
   - Cifrado: Sí

2. **Google AdMob** (Publicidad)
   - Datos compartidos: ID del dispositivo, datos de uso
   - Propósito: Mostrar anuncios personalizados
   - Cifrado: Sí

### Prácticas de seguridad

```
☑️ Los datos se cifran en tránsito (HTTPS/TLS)
☑️ Los datos se cifran en reposo
☑️ Los usuarios pueden solicitar la eliminación de datos
☑️ Cumplimiento con GDPR y políticas de privacidad
```

### Formulario de seguridad de datos (resumen)

| Tipo de dato | Recopilado | Compartido | Propósito | Opcional |
|--------------|------------|------------|-----------|----------|
| Email | ✓ | ✓ (Supabase) | Autenticación | No |
| Nombre | ✓ | ✓ (Supabase) | Perfil | No |
| Fotos | ✓ | ✓ (Supabase) | Compartir con pareja | Sí |
| Mensajes | ✓ | ✓ (Supabase) | Chat | Sí |

## 🚀 Pasos para publicar

1. **Crear cuenta de Google Play Console**
   - Pago único de $25 USD
   - Verificación de identidad

2. **Crear nueva aplicación**
   - Seleccionar idioma predeterminado
   - Nombre de la app
   - Tipo: App

3. **Completar todas las secciones**
   - Seguir esta guía paso a paso
   - Marcar cada checklist

4. **Subir el APK/AAB**
   - Ir a "Producción" o "Prueba interna"
   - Subir archivo firmado
   - Completar notas de la versión

5. **Enviar para revisión**
   - Revisar todos los campos
   - Enviar para aprobación
   - Tiempo de revisión: 1-7 días
